# .ai Context

This directory is the fast-path context bundle for contributors who need to get productive without reverse-engineering the repository from scratch.

## Files

- `repo-context.md`: architecture, boundaries, and important paths
- `junior-onboarding.md`: first-pass onboarding for less experienced contributors
- `testing-methodology.md`: what to test, what not to over-test, and how CI fits in
- `task-intake.md`: how to frame new work before writing code
- `current-milestone.md`: mirrored active milestone summary

Use this directory as operational context. The canonical product and planning records still live under `docs/`.
