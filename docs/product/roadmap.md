# Roadmap

## Near Term

- harden unsupported-state UX and contributor onboarding
- stabilize CI diagnostics and artifact retention
- validate the live capture path on physical TrueDepth hardware
- add a second deformation module

## Medium Term

- add save and load flows for mesh sessions
- support before and after comparison
- improve deformation parameter handling and sequencing
- expand integration coverage around app state transitions

## Longer Term

- support additional capture inputs beyond TrueDepth
- improve mesh quality, smoothing, and landmark handling
- add richer export and offline-processing workflows
- evaluate data-driven or ML-assisted deformation modules

## Guardrails

- do not trade architectural clarity for short-term demos
- do not market the current system as medically accurate
- do not let UI or rendering concerns leak back into capture and simulation boundaries
