# Delivery Model

## Planning Style

This repository uses a hybrid milestone model.

- milestones define the active engineering objective and exit criteria
- milestone documents may contain sprint-like checklists when short-term coordination is useful
- backlog items stay separate from the current milestone so priorities remain explicit

## Why This Model

- the project is early enough that rigid sprint ceremony would add overhead
- the architecture is still evolving, so milestone scope matters more than fixed dates
- contributors still need a stable place to understand what is active now versus later

## Definition Of Done

Work is considered done for a milestone when:

- code changes are merged
- affected docs are updated
- tests are added or adjusted where appropriate
- CI passes
- known runtime limitations are documented instead of hidden
